## Deterministic rule-hardening tests

The v0.3.36 rule-hardening slice adds deterministic tests for false-PASS prevention, remote-evidence truthfulness, direct `ns -> tools/ns_*.sh` adapter reintroduction, and stale shell-file test expectations.

## Terminal-log mandate hardening

The next standard-error hardening step is to route non-trivial local mutation blocks through a log-backed runner or generated work-order path so `terminal_log=NONE` cannot silently return for mutation work.

## Summary format and execution-origin hardening

The final summary now records the slice name, execution origin, state mode, mode-check result, and a deterministic local/remote switch hint. This separates where work was produced from whether evidence is remote-readable.

## Logged-block status propagation hardening

The standard-error hardening baseline now treats STATUS-losing pipeline/subshell logging constructs as unsafe for local mutation blocks. Use a runner, command report, or file-descriptor log capture that preserves shell state.


### Summary renderer usage hardening

- Use `removed legacy-wrapper route summary` for final run summaries in new local mutation blocks.
- Do not handwrite large final-summary `printf` blocks in new runbooks.
- Distinguish `terminal_log_remote` from `terminal_log_local` in every final summary.
- On FAIL with no committed remote log, the next reply remains `f` with pasted output.
