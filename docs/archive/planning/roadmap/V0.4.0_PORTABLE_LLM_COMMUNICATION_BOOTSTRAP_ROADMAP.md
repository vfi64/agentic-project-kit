# v0.4.0 Portable LLM Communication Bootstrap Roadmap

Status: active
Decision status: accepted
Review policy: review before further GUI expansion

Status-date: 2026-05-22
Priority: highest before further GUI-MVP feature expansion
## Roadmap goal
Make the agentic-project-kit safe for large-LLM assisted development by moving communication, summary, evidence, bootstrap, drift detection, and handoff behavior from chat convention into repository-owned, portable, deterministic, test-backed mechanisms.
## Strategic direction
Long prompts alone are not enough. The durable design is:
1. short mandatory bootstrap instruction,
2. machine-readable compiled context,
3. normative governance documents,
4. portable Python checks,
5. one summary renderer and one vocabulary,
6. deterministic drift detection,
7. generated handoff prompt when drift is detected.
## Milestones
### M1: Planning and roadmap recorded
Deliverables:
- `docs/planning/V0.4.0_PORTABLE_LLM_COMMUNICATION_BOOTSTRAP_PLAN.md`
- this roadmap
Acceptance:
- The top priority is visible before implementation starts.
- The plan explicitly covers communication rules, summary structure, portable execution, drift detection, and handoff prompt generation.
### M2: Governance documents added
Deliverables:
- `docs/governance/CHAT_COMMUNICATION_CONTRACT.md`
- `docs/governance/PORTABLE_CHAT_EXECUTION_CONTRACT.md`
- `docs/governance/CHAT_BOOTSTRAP_AND_DRIFT_CONTRACT.md`
- updates to `docs/governance/FINAL_SUMMARY_CONTRACT.md`

- Successor chats know what to read first.

- Terminal-to-LLM and LLM-to-terminal communication rules are explicit.

### M3: Machine-readable context and coverage updated
Deliverables:
- `.agentic/compiled_agent_context.yaml` includes mandatory bootstrap sources and forbidden patterns.
- `docs/DOCUMENTATION_COVERAGE.yaml` protects the new anchors.
- `docs/STATUS.md` and `docs/handoff/CURRENT_HANDOFF.md` point to the canonical contracts without duplicating them excessively.
Acceptance:
- Drift-prone rules are not only prose in a chat or roadmap.
- `check-docs` can detect missing anchors.
### M4: Summary renderer alignment
Deliverables:
- `run_summary_renderer.py` aligned with the final summary contract.
- tests for valid and invalid result values.
- no final `REMOTE_EVIDENCE: PENDING`, `NONE`, or `CHAT_ONLY` accepted.
- `NO-COMMAND` supported where declared.
Acceptance:
- One vocabulary is shared by docs, renderer, and tests.
- Validation errors name invalid values precisely.
### M5: Portable communication and bootstrap checks
Deliverables:
- `agentic-kit bootstrap-check`
- `agentic-kit comm-rules-check`
- tests for both commands
Acceptance:
- Checks use Python APIs and `shell=False` where subprocesses are needed.
- Checks work across macOS, Linux, and Windows assumptions.
- Missing required sources or contract drift fails deterministically.
### M6: Drift handoff prompt path
Deliverables:
- `agentic-kit handoff-prompt --reason drift` or equivalent existing handoff command extension.
- test ensuring mandatory source list, drift findings, forbidden patterns, and next safe step appear.
Acceptance:
- When drift is detected, the system can produce a successor-chat prompt without relying on memory.
### M7: Integration gates and closeout
Deliverables:
- targeted tests,
- full pytest,
- ruff,
- check-docs,
- doctor,
- status/handoff freshness if available,
- remote evidence log.
Acceptance:
- The roadmap can move from in-progress to active baseline only after gates pass.
## Optimization principles for every implementation step
Each step must ask and document:
1. Can this rule be expressed once and referenced instead of duplicated?
2. Can a Python check replace shell assumptions?
3. Can a test catch the exact regression we just observed?
4. Can the LLM have less freedom and more deterministic input?
5. Can a failed or partial run still produce honest evidence without relabeling failure as success?
## Work pause rule
Further GUI feature expansion is paused until at least M2-M4 are complete and M5 has a minimal passing implementation or documented blocker. GUI verification may continue only as bounded evidence work when it does not weaken this priority.
