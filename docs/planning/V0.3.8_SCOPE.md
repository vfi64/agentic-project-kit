# v0.3.8 Scope Plan

Status-date: 2026-05-15
Status: proposed planning slice

## Recommended release focus

v0.3.8 should continue the Guided CLI usability line instead of starting a broad Pattern Advisor implementation.

The preferred scope is Guided CLI Usability v2: make the workflow commands easier to use safely after v0.3.7 completed the first read-only status explanation layer.

## Why this is the preferred next slice

v0.3.7 closed Guided Workflow Usability v1 with:

- read-only `agentic-kit workflow status --explain`;
- explicit safety wording;
- `current_report` explanation;
- README quick command guide;
- guided workflow state compass.

The next release should build directly on that usable surface. This keeps the project focused on concrete daily workflow improvements instead of introducing a larger advisory subsystem too early.

## Candidate scope for Guided CLI Usability v2

A small v0.3.8 slice may include one or more of:

- clearer `workflow status --explain` wording for `IDLE`, `REQUESTED`, `UPLOADED`, `FAILED`, and dirty working trees;
- a read-only workflow guidance command if it is strictly bounded;
- improved documentation for the `request -> run -> review -> cleanup` loop;
- tests that lock down safety wording and stop-state behavior.

## Explicit non-goals

v0.3.8 should not include:

- Pattern Advisor MVP implementation;
- automatic project decisions;
- semantic recommendation engines;
- new unbounded repair behavior;
- changes to release automation unless directly required by the selected CLI usability slice.

## Pattern Advisor position

`docs/ideas/PATTERN_ADVISOR.md` remains a non-binding idea note. It can inform later design, but it should not drive v0.3.8 unless a separate, contract-first Pattern Advisor MVP is explicitly approved.

## Acceptance criteria for a v0.3.8 implementation slice

Before implementation starts, the selected slice should have:

- a one-paragraph user-facing goal;
- an explicit command-level contract;
- deterministic tests;
- no hidden state mutation in read-only guidance paths;
- updated README and workflow documentation;
- green `pytest`, `ruff`, `check-docs`, `doctor`, and `doc-mesh-audit`.
